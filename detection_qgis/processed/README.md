# 🌙 Lunar Analysis – Processed Outputs

This directory contains the generated outputs and runnable pipeline for the QGIS-based lunar terrain analysis. Results here are consumed by the backend server and can be inspected or post-processed directly.

---

## 📁 Contents

- `lunar_main.py` – Orchestrates the full terrain analysis workflow (invoked by backend or QGIS Python)
- `lunar_risk_analysis_results.json` – Consolidated JSON (large; 100MB+) with composite and component metrics
- `comprehensive_lunar_risk_report.txt` – Human-readable summary of composite risk
- `json_results/` – Individual JSON outputs per component/module (preferred for programmatic consumption)
- `counter_outputs/`, `hillshade_outputs/`, `crater_walls/`, `headwalls_scraps/`, `debris_path_output/`, `Terrian_Reggedness_output/` – Intermediate/final artifacts per analysis track (rasters, text reports, logs)
- Component scripts (runnable standalone as needed):
  - `slope.py`, `hillshade.py`, `counter.py`, `Terrain_Ruggedness.py`, `Debris_path.py`, `scraps_headwalls.py`, `crater_edges.py`, `curvature_statistics.py`, `elevation_statistics.py`, `tif_processor.py`
- Utilities:
  - `test_large_file_optimization.py` – Helpers/experiments for working with large JSON files
  - `.gitignore` – Keeps bulky intermediates out of version control

> Note: Folder name `Terrian_Reggedness_output` is preserved as generated by the pipeline.

## 🚀 Running the Pipeline

### Option A – Via Backend API (recommended)
The backend server triggers this pipeline using the QGIS Python launcher and returns aggregated outputs:
- `POST /api/lunar-analysis` (see backend `README.md`), optional body:
  - `{ "dem_path": "C:\\path\\to\\your.tif" }`
- Results are read from `detection_qgis/processed/json_results` and consolidated for the response.

### Option B – Manual (Windows, QGIS installed)
Run with QGIS’ Python (paths may vary):

```powershell
# Example launcher used by the backend
"C:\Program Files\QGIS 3.40.9\bin\python-qgis-ltr.bat" "lunar_main.py" "C:\path\to\your.tif"
```

Outputs will be written into the subfolders here, and JSON summaries into `json_results/` and/or the top-level JSON files.

### Option C – Module-by-Module
You can execute individual component scripts (e.g., `slope.py`, `Terrain_Ruggedness.py`) for ad‑hoc runs. Ensure required dependencies (GDAL/QGIS stack or raster libs) are available and inputs are correctly configured in each script.

## 📦 Outputs & Where to Look

- High-level summary:
  - `comprehensive_lunar_risk_report.txt` – concise, human-readable overview
- Machine-readable data:
  - `json_results/` – preferred for integration; multiple JSON files per component/step
  - `lunar_risk_analysis_results.json` – single large file aggregating results (see size tips below)
- Component-specific reports:
  - Under `*_outputs/` folders; e.g., `counter_outputs/lunar_contour_analysis_report.txt`

---

## 🔗 Backend Integration

- The backend endpoint `POST /api/lunar-analysis` runs `lunar_main.py` and aggregates JSON from:
  - `detection_qgis/processed/json_results`
- If you add new result JSONs, write them into `json_results/` to have them included automatically in responses.

---

## ⚙️ Large File Tips (125MB+ JSON)

- Prefer consuming individual files from `json_results/` rather than `lunar_risk_analysis_results.json`.
- Use streaming reads or chunked processing when you must read the large file.
- Consider compressing (`.gz`) for storage/transport.
- The utility `test_large_file_optimization.py` contains examples for handling large JSON efficiently.

---

## 🧭 Conventions & Requirements

- Rasters should share CRS, extent, and resolution for consistent metrics.
- Paths with spaces are supported via proper quoting (see backend command construction).
- Ensure QGIS/GDAL environment is available for raster processing scripts.

---

## 🧪 Validation Checklist

- Verify expected files appear under `json_results/` after a run
- Check `comprehensive_lunar_risk_report.txt` for a reasonable composite score
- Inspect component reports under `*_outputs/` for anomalies (NaNs, empty layers)

---

## 🛠 Troubleshooting

- QGIS launcher path invalid → update the path and retry
- Missing outputs → confirm input DEM exists and scripts have permissions
- Backend cannot find results → ensure JSONs are under `processed/json_results`
- Slow/large runs → expect multi‑hour processing for multi‑GB DEMs; watch progress where available
